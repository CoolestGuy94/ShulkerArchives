Using NBT Data:
A Guide

Beginner

So, you came here, found a cool Kit or Item, and want to get it in Minecraft, but don't know how.

That's where this guide comes in!

In the following section, we'll look at the basics of using NBT Data to get cool Kits and Items in Minecraft, including the tools and commands needed.

Setup

The easiest way to add NBT in-game, is via a command. However, the command can be complicated or even impossible if you play Vanilla and/or have a really long string of NBT; For that reason, it's best to use a Modded setup for this.

Below is the recommended group of Mods you will need to start off

  • Fabric
  • Fabric API
  • Meteor Client

There are fancier editors such as IBE, but they are far more complicated to use.

Once you have loaded into your world (or a server where you have sufficient privileges), take the item or container indicated in the archive, such as a red_shulker_box or diamond_sword, and hold it.

Next, open chat, and type .nbt set followed by the NBT copied from the archive.

If the text is all blue, then there are no issues, and you can go ahead an enter it. Enjoy your newly minted Kit or Item!

If however the text is all red, then there is an issue with the command or NBT, and you need to begin again.

Should the text be all red even if you followed all steps correctly, please open an Issue using the Changes & Issues template, stating which Kit or Item you were attempting to make, and we'll see if there's an issue with the archived NBT.

Known Bugs/Issues

Below are known bugs/issues related to NBT Editing, and any known solutions for them.


oshi.architecture.properties

This error (seen in log files following a crash related to NBT) is basically Fabric shitting the bed over something in the NBT, despite there being no real issue.

The major downside to this bug is that it can prevent loading certain Kits and Items in a Minecraft instance that is using Fabric.

A common workaround is to load the Kits/Items in a Forge instance, then copy them over to fabric via a world save file.
